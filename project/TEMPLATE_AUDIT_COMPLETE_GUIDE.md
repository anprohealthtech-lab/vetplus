# Template Audit System - Complete Validation Guide

## Overview

The Template Audit system has been enhanced to validate **ALL critical placeholder categories** in laboratory report templates:

1. ✅ **Patient Metadata** - Patient information and order details
2. ✅ **Test/Result Values** - Analyte placeholders with flags
3. ✅ **Approval/Signature** - Approver information and signatures  
4. ✅ **Section Content** - Doctor-filled content areas (NEW)
5. ✅ **Placeholder Quality** - Deprecated, unknown, duplicate checks

---

## What Gets Audited

### 1. Patient Metadata Validation

**Checks:**
- Patient metadata table exists in template
- Required patient fields are present

**Required Placeholders:**
- `{{patientName}}` ✓
- `{{patientAge}}` ✓
- `{{patientGender}}` ✓
- `{{patientId}}` ✓
- `{{registrationDate}}` ✓
- `{{locationName}}` ✓
- `{{sampleCollectedAt}}` ✓
- `{{approvedAt}}` ✓
- `{{referringDoctorName}}` ✓
- `{{orderId}}` ✓

**Status:**
- ✓ **PASS**: Table present with all required fields
- ⚠ **ATTENTION**: Table present but some fields missing
- ✗ **FAIL**: Patient metadata table not found

---

### 2. Test/Result Validation (CRITICAL)

**Checks:**
- All analytes from linked test group are in template
- Each test **VALUE** placeholder has corresponding **FLAG** placeholder
- No unknown test parameters referenced

**Example:**
```html
<!-- ✓ CORRECT -->
<td>{{Hemoglobin}}</td>
<td>{{Hemoglobin_flag}}</td>

<!-- ✗ WRONG - Missing flag! -->
<td>{{Hemoglobin}}</td>
<td>-</td>
```

**Flags Generated By:**
- AI-powered result analysis during verification
- Values: `"high"`, `"low"`, `"normal"`, or empty
- Stored in `result_values.flag` (10 char limit)

**Status:**
- ✓ **PASS**: All tests have both value AND flag placeholders
- ⚠ **ATTENTION**: Some tests missing flags or analytes not in template
- ✗ **FAIL**: Many missing flags or wrong test parameters

---

### 3. Section Content Validation (NEW ✨)

**Checks:**
- Presence of section content placeholders (doctor's notes)
- Detection of deprecated section placeholders
- Recommendations for adding section areas

**Valid Section Placeholders:**
- `{{impression}}` - Interpretation summary (most common)
- `{{findings}}` - Observations and findings
- `{{conclusion}}` - Final conclusion
- `{{recommendation}}` - Treatment suggestions
- `{{clinical_history}}` - Patient history notes
- `{{technique}}` - Methodology description
- `{{comments}}` - Additional notes

**Deprecated (Must Replace):**
- ❌ `{{interpretation_summary}}` → Use `{{impression}}` instead

**Why This Matters:**
Section content is stored in `result_section_content` table, mapped by `lab_template_sections.placeholder_key`. If you use the wrong placeholder key, the doctor's notes won't appear in PDFs!

**Status:**
- ✓ **PASS**: Has at least one valid section placeholder
- ⚠ **ATTENTION**: No section placeholders OR deprecated ones found
- ✗ **FAIL**: Only deprecated placeholders present

---

### 4. Approval/Signature Validation (NEW ✨)

**Checks:**
- Approver name is present (REQUIRED)
- Approver role/title is present (recommended)
- Signature image placeholder present (optional)
- Detection of deprecated approval placeholders

**Valid Approval Placeholders:**
- `{{approverName}}` ✓ - Approver's full name (REQUIRED)
- `{{approvedByName}}` ✓ - Alias for approverName (REQUIRED)
- `{{approverRole}}` ⚠ - Approver's role/designation (recommended)
- `{{approverSignature}}` ⚠ - Signature image URL (optional)

**Deprecated (Must Replace):**
- ❌ `{{signatoryName}}` → Use `{{approverName}}` instead
- ❌ `{{signatoryTitle}}` → Use `{{approverRole}}` instead

**Common Signature Block Pattern:**
```html
<div class="signature-block">
  <img src="{{approverSignature}}" alt="Signature" />
  <p><strong>{{approverName}}</strong></p>
  <p>{{approverRole}}</p>
</div>
```

**Status:**
- ✓ **PASS**: Has approver name, no deprecated placeholders
- ⚠ **ATTENTION**: Missing approver role OR has deprecated placeholders
- ✗ **FAIL**: No approver information at all

---

### 5. Placeholder Quality Checks

**Checks:**
- Unknown placeholders (possible typos)
- Duplicate placeholders
- Deprecated placeholders across all categories
- Missing required vs available placeholders

**What's Ignored (Not Audited):**
The audit **DOES NOT** flag these as errors (they're optional):
- Header/footer placeholders (`group="header"`, `group="footer"`)
- Lab information (`group="lab"`)
- Branding assets (`group="branding"`)
- Header/footer images
- Signature blocks (structure)

**Only These Groups Are Audited:**
- ✓ `group="patient"`
- ✓ `group="test"`
- ✓ `group="signature"`
- ✓ `group="section"` (NEW)

## Audit Status Definitions

### ✓ PASS
**All conditions met:**
- Patient metadata table present with all fields
- All test values have corresponding flag placeholders
- At least one approval placeholder exists (name)
- No deprecated placeholders used
- Analyte coverage matches test group

### ⚠ ATTENTION
**Minor fixable issues:**
- Missing optional analytes or section placeholders
- Some tests missing flag placeholders
- Deprecated placeholders present (needs replacement)
- Unknown placeholders (possible typos)
- Missing optional approval fields (role, signature)

### ✗ FAIL
**Critical issues preventing proper PDF generation:**
- Patient metadata table completely missing
- Required patient/order placeholders absent
- No approval information present
- Many test values without flags
- Wrong test group linked

## How to Use

### 1. Run Audit in Template Studio

1. Open Template Studio (CKEditor)
2. Load your template
3. Link to appropriate test group
4. Click **"Audit"** button
5. Review results in modal

### 2. Review Audit Results

The modal shows categorized results:

**Patient Metadata**
- ✓ Table detected: Yes
- Missing columns: (none)

**Section Content (Doctor Notes)**
- ✓ Section placeholders found: {{impression}}, {{findings}}
- ✗ Deprecated: {{interpretation_summary}} → Use {{impression}}

**Approval & Signature**
- ✓ Approver Name: Present
- ⚠ Approver Role: Optional (add {{approverRole}})
- ✓ Signature Image: Present
- ✗ Deprecated: {{signatoryName}} → Use {{approverName}}

**Placeholders**
- ✗ Missing flag placeholders: {{Hemoglobin_flag}}, {{Platelet_flag}}
- ✗ Deprecated: {{interpretation_summary}}, {{signatoryName}}

**Analyte Coverage**
- ✓ Template and test group are aligned

**Recommendations**
- Replace {{interpretation_summary}} with {{impression}}
- Replace {{signatoryName}} with {{approverName}}

**Option A: Manual Fix**
1. Edit template in CKEditor
2. Replace deprecated placeholders
3. Add missing flag placeholders
4. Save template

**Option B: AI Auto-Fix**
1. Click **"Implement Changes"** button
2. AI analyzes audit results
3. AI generates patched HTML
4. Review changes in editor
5. Save if satisfied or click "Revert AI Changes"

## Common Issues & Fixes

### Issue 1: "Missing flag placeholders"
**Example:** Template has `{{Hemoglobin}}` but not `{{Hemoglobin_flag}}`

**Fix:**
```html
<!-- Before -->
<tr>
  <td>Hemoglobin</td>
  <td>{{Hemoglobin}}</td>
  <td>g/dL</td>
  <td>13.5-17.5</td>
  <td>-</td> <!-- ❌ Hardcoded -->
</tr>

<!-- After -->
<tr>
  <td>Hemoglobin</td>
  <td>{{Hemoglobin}}</td>
  <td>g/dL</td>
  <td>13.5-17.5</td>
  <td>{{Hemoglobin_flag}}</td> <!-- ✓ Dynamic -->
</tr>
```

### Issue 2: "Deprecated section placeholder"
**Example:** Template uses `{{interpretation_summary}}`

**Fix:**
```html
<!-- Before -->
<h3>Interpretation Summary</h3>
<p>{{interpretation_summary}}</p> <!-- ❌ Deprecated -->

<!-- After -->
<h3>Impression</h3>
<p>{{impression}}</p> <!-- ✓ Current -->
```

### Issue 3: "Deprecated approval placeholder"
**Example:** Template uses `{{signatoryName}}`

**Fix:**
```html
<!-- Before -->
<div class="signature">
  <p>{{signatoryName}}</p> <!-- ❌ Deprecated -->
  <p>{{signatoryTitle}}</p> <!-- ❌ Deprecated -->
</div>

<!-- After -->
<div class="signature">
  <p>{{approverName}}</p> <!-- ✓ Current -->
  <p>{{approverRole}}</p> <!-- ✓ Current -->
</div>
```

### Issue 4: "No section content placeholders"
**Fix:** Add at least one section area:

```html
<section class="interpretation">
  <h3>Impression</h3>
  <p>{{impression}}</p>
</section>

<section class="findings">
  <h3>Findings</h3>
  <p>{{findings}}</p>
</section>
```

### Issue 5: "No approval information"
**Fix:** Add signature block:

```html
<div class="approval-section">
  <img src="{{approverSignature}}" alt="Signature" style="height: 60px;" />
  <p><strong>{{approverName}}</strong></p>
  <p>{{approverRole}}</p>
  <p><small>Approved at: {{approvedAt}}</small></p>
</div>
```

## Best Practices

### ✅ DO:
1. **Run audit BEFORE marking template as default**
2. **Fix all "fail" issues immediately**
3. **Address "attention" issues for production templates**
4. **Use current placeholder names** (not deprecated ones)
5. **Include flag placeholders for every test value**
6. **Add at least one section content area** ({{impression}})
7. **Include approval information** ({{approverName}})
8. **Link template to correct test group**
9. **Re-run audit after making changes**
10. **Test with actual patient data** using Preview

### ❌ DON'T:
1. **Don't ignore deprecated placeholder warnings**
2. **Don't hardcode flag values** (use `{{ParameterName_flag}}`)
3. **Don't use `{{interpretation_summary}}`** (use `{{impression}}`)
4. **Don't use `{{signatoryName}}`** (use `{{approverName}}`)
5. **Don't skip linking to test group**
6. **Don't mark as default without passing audit**
7. **Don't ignore missing flag placeholders** (AI won't work without them)
8. **Don't mix old and new placeholder names**

### Database Structure

**Section Content Flow:**
```
lab_template_sections (defines available sections)
  → placeholder_key (e.g., "impression")
  
result_section_content (doctor fills during verification)
  → section_id (links to lab_template_sections)
  → final_content (actual doctor's notes)
  
get_report_template_context() function
  → joins tables, maps by placeholder_key
  → returns sectionContent: { "impression": "doctor's notes here" }
  
PDF generation
  → Nunjucks replaces {{impression}} with actual content
```

## Migration Path for Existing Templates

If you have templates created before this enhancement:

### Step 1: Identify Deprecated Placeholders
Run audit on each template to identify:
- `{{interpretation_summary}}` usage
- `{{signatoryName}}` usage
- `{{signatoryTitle}}` usage
- Missing flag placeholders

### Step 2: Create Replacement Map
```javascript
const replacements = {
  '{{interpretation_summary}}': '{{impression}}',
  '{{signatoryName}}': '{{approverName}}',
  '{{signatoryTitle}}': '{{approverRole}}',
};
```

### Step 3: Update Templates
For each template:
1. Open in Template Studio
2. Run audit
3. Click "Implement Changes" (AI auto-fix)
4. OR manually replace using Find & Replace
5. Add missing flag placeholders
6. Re-run audit until PASS
7. Save and mark as updated

### Step 4: Update Database
Ensure `lab_template_sections` has correct placeholder keys:
```sql
UPDATE lab_template_sections 
SET placeholder_key = 'impression'
WHERE placeholder_key = 'interpretation_summary';
```

### Step 5: Verify PDFs
Generate test PDFs to ensure:
- Section content appears correctly
- Approver information displays
- Flags show properly (not "-")

## Support & Troubleshooting

### Audit not running?
- Check Gemini API key is configured
- Verify template is linked to test group
- Check browser console for errors

### False positives?
- Audit only checks patient/test/signature/section groups
- Header/footer/lab placeholders are intentionally ignored
- Optional placeholders show as "attention" not "fail"

### AI fixes not working?
- Try manual edits first
- Use "Revert AI Changes" if needed
- Re-run audit after manual fixes

### Section content not appearing in PDF?
1. Check `lab_template_sections.placeholder_key` matches template
2. Verify doctor filled content during verification
3. Run SQL: `SELECT * FROM result_section_content WHERE result_id = 'xxx'`
4. Check `get_report_template_context()` returns sectionContent
