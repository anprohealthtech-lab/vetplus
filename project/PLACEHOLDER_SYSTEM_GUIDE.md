# LIMS v2 - Placeholder System Guide

## Overview

The LIMS v2 Template Studio uses a comprehensive placeholder system to dynamically inject data into PDF reports. Placeholders are double-curly-braced tokens like `{{patientName}}` that get replaced with actual values during PDF generation.

---

## Available Placeholder Categories

### 1. **Patient & Order Placeholders** (`patient` group)

Patient-specific and order-related data that comes from the order and patient records.

| Placeholder | Description | Example Value |
|------------|-------------|---------------|
| `{{patientName}}` | Full patient name | "John Doe" |
| `{{patientAge}}` | Patient age in years | "45" |
| `{{patientGender}}` | Patient gender | "Male" |
| `{{patientDOB}}` | Date of birth | "1979-03-15" |
| `{{patientId}}` | Internal patient UUID | "abc-123-def" |
| `{{patientDisplayId}}` | Human-readable patient ID | "16-Dec-2025-1" |
| `{{referringDoctorName}}` | Referring physician | "Dr. Sarah Wilson" |
| `{{registrationDate}}` | Patient registration date | "2025-12-16" |
| `{{locationName}}` | Collection center/location | "Main Lab" |
| `{{sampleCollectedAt}}` | Sample collection timestamp | "2025-12-16T10:30:00Z" |
| `{{sampleCollectedBy}}` | Phlebotomist name | "Tech John" |
| `{{approvedAt}}` | Approval/verification time | "2025-12-16T14:45:00Z" |
| `{{orderId}}` | Order UUID | "xyz-789-abc" |
| `{{orderNumber}}` | Human-readable order number | "ORD-2025-001" |

### 2. **Lab Information Placeholders** (`lab` group)

Lab-level configuration and branding information.

| Placeholder | Description | Example Value |
|------------|-------------|---------------|
| `{{labName}}` | Laboratory name | "MediLab Diagnostics" |
| `{{labAddress}}` | Lab address | "123 Health St, Medical District" |
| `{{labPhone}}` | Lab phone number | "+91 80 1234 5678" |
| `{{labEmail}}` | Lab email | "reports@medilab.com" |
| `{{labWebsite}}` | Lab website URL | "https://medilab.com" |
| `{{labLogoUrl}}` | Lab logo image URL | "https://cdn.lab.com/logo.png" |
| `{{labRegistrationNumber}}` | Lab license/registration | "LAB-2024-001" |

---

### 3. **Test Result Placeholders** (`test` group)

Dynamic placeholders based on the test group linked to the template. Each analyte in the test group gets two placeholders:

**Format:**
- **Value**: `{{AnalyteName}}` (e.g., `{{Hemoglobin}}`)
- **Flag**: `{{AnalyteName_flag}}` (e.g., `{{Hemoglobin_flag}}`)

**Example for CBC:**
| Placeholder | Description | Example Value |
|------------|-------------|---------------|
| `{{Hemoglobin}}` | Hemoglobin result value | "14.5" |
| `{{Hemoglobin_flag}}` | Hemoglobin flag | "normal" / "high" / "low" |
| `{{Platelet}}` | Platelet count | "250000" |
| `{{Platelet_flag}}` | Platelet flag | "low" |
| `{{WhiteBloodCellCount}}` | WBC result | "7.5" |
| `{{WhiteBloodCellCount_flag}}` | WBC flag | "normal" |

**⚠️ CRITICAL**: Always include BOTH the value placeholder AND the `_flag` placeholder for each test parameter. Flags are generated by AI during result verification.

---

### 4. **Section Content Placeholders** (`section` group)

Doctor-filled content sections that are populated during result verification. These are stored in the `result_section_content` table and mapped by `placeholder_key` from `lab_template_sections`.

| Placeholder | Description | Typical Use |
|------------|-------------|-------------|
| `{{impression}}` | Impression/Interpretation summary | Doctor's overall assessment |
| `{{findings}}` | Findings/Observations | Detailed observations from analysis |
| `{{conclusion}}` | Conclusion | Final diagnostic conclusion |
| `{{recommendation}}` | Recommendations/Suggestions | Treatment or follow-up suggestions |
| `{{clinical_history}}` | Clinical History | Patient history relevant to test |
| `{{technique}}` | Technique/Methodology | Description of testing methodology |
| `{{comments}}` | Comments/Notes | Additional notes from the lab |

**Example Content:**
```
{{impression}} might contain:
"Red blood cells show mild hypochromia with occasional target cells. 
No significant abnormality detected."
```

**⚠️ NOTE**: The placeholder key in `lab_template_sections` must match the placeholder used in the template. For example:
- Database record: `placeholder_key = 'impression'`
- Template usage: `{{impression}}`

**DO NOT USE:** `{{interpretation_summary}}` (not in database) - Use `{{impression}}` instead.

---

### 5. **Signatory/Approver Placeholders** (`signature` group)

Information about the person who approved/verified the report.

| Placeholder | Description | Example Value |
|------------|-------------|---------------|
| `{{approverSignature}}` | Approver's signature image URL | "https://cdn.lab.com/sig.png" |
| `{{approverName}}` | Approver's full name | "Dr. Sarah Johnson" |
| `{{approvedByName}}` | Same as approverName (alias) | "Dr. Sarah Johnson" |
| `{{approverRole}}` | Approver's role/title | "Medical Director" |

**Common Usage Pattern:**
```html
<div class="signatory-block">
  <img src="{{approverSignature}}" alt="Signature" style="height: 60px;" />
  <p><strong>{{approverName}}</strong></p>
  <p>{{approverRole}}</p>
</div>
```

---

### 6. **Branding Asset Placeholders** (`branding` group)

Lab-uploaded branding assets (headers, footers, logos, etc.) from the `lab_branding_assets` table.

These are dynamically generated based on uploaded assets and appear in the picker when assets are marked as default.

## How the System Works

### 1. **Template Studio**
- Open Template Studio (CKEditor version)
- Click "Placeholders" button
- Browse categories: Lab, Patient, Test Group, Section, Signature, Branding
- Click a placeholder to select it
- Click "Copy Placeholder" or "Insert into Editor"

### 2. **Multi-Select Copy** (NEW FEATURE ✨)
- Click "Multi-Select" button in Placeholder Picker
- Checkboxes appear next to each placeholder
- Select multiple placeholders
- Click "📋 Copy X Selected" to copy all to clipboard (one per line)
- Paste into template as needed

### 3. **PDF Generation Flow**
1. Order approved → triggers PDF generation
2. System calls `get_report_template_context(order_id)` SQL function
3. Function builds JSONB context with all placeholder values:
   ```json
   {
     "patientName": "John Doe",
     "patientAge": 45,
     "analytes": [
       { "parameter": "Hemoglobin", "value": "14.5", "flag": "normal" }
     ],
     "sectionContent": {
       "impression": "Findings indicate normal blood work..."
     },
     "placeholderValues": { /* flattened map */ }
   }
   ```
4. Nunjucks template engine replaces `{{placeholder}}` tokens
5. HTML sent to Puppeteer Service → generates PDF

## Best Practices

### ✅ DO:
- Always use BOTH value and `_flag` placeholders for test results
- Use `{{impression}}` for doctor's interpretation summary
- Check that `lab_template_sections.placeholder_key` matches your template
- Include signatory block with signature image + name + role
- Test template with "Preview" before marking as default

### ❌ DON'T:
- Don't hardcode values like "-" in flag columns
- Don't use `{{interpretation_summary}}` (not in database, use `{{impression}}`)
- Don't use `{{signatoryName}}` or `{{signatoryTitle}}` (use `{{approverName}}` and `{{approverRole}}`)
- Don't forget to link template to correct test group

## Common Issues & Fixes

### Issue: "Flags showing as '-' in PDF"
**Cause**: Template has hardcoded `-` instead of `{{ParameterName_flag}}`  
**Fix**: Replace `<td>-</td>` with `<td>{{Hemoglobin_flag}}</td>`

### Issue: "Section content not appearing"
**Cause 1**: Wrong placeholder key  
**Fix**: Check `lab_template_sections` table, use the exact `placeholder_key` value

**Cause 2**: No content saved  
**Fix**: Doctor must fill section during verification in Results → AI Tools

### Issue: "Signatory name not showing"
**Cause**: Using wrong placeholder  
**Fix**: Use `{{approverName}}` not `{{signatoryName}}`

## Template Audit System

The Template Audit function validates templates automatically:

1. **Checks for required placeholders**: patientName, orderId, etc.
2. **Verifies test group analyte coverage**: All test parameters must have placeholders
3. **Validates flag placeholders**: Each test value must have corresponding `_flag`
4. **Identifies unknown placeholders**: Warns about typos or deprecated keys

**To run audit**: Click "Audit" button in Template Studio

---

## Database Structure

### Key Tables:
- `lab_templates`: Stores template HTML/CSS
- `lab_template_sections`: Defines available section placeholders
- `result_section_content`: Doctor-filled section content
- `result_values`: Test result values and AI-generated flags
- `lab_branding_assets`: Uploaded branding assets

### Key Function:
```sql
get_report_template_context(order_id UUID) RETURNS JSONB
```
Builds the complete context with all placeholder values for an order.

## Support

If placeholders are not working:
1. Check Template Audit for errors
2. Verify test group is linked to template
3. Confirm section `placeholder_key` matches template usage
4. Check database context with: `SELECT get_report_template_context('order-id-here')`

For multi-select issues, ensure you're using Template Studio CKE (not GrapesJS version).
